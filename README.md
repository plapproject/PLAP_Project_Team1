# DonkeyCar Data Manager

> DonkeyCar Tub 데이터를 불러오고, 잘못된 프레임을 검토/제외/복원한 뒤, 원본을 보존한 상태로 학습용 Clean 데이터셋을 만드는 C# WinForms 기반 데이터 관리 도구입니다.

## 프로젝트 방향

이 프로젝트의 핵심은 **자율주행 모델을 직접 잘 만들기**가 아니라, 모델이 학습할 데이터를 더 쉽게 확인하고 정리하는 것입니다.

DonkeyCar 주행 중에는 다음과 같은 문제가 자주 생깁니다.

- 벽 충돌 직전/직후 프레임
- 차선 이탈 프레임
- 이미지 상황과 조향값이 맞지 않는 프레임
- 정지에 가까운 속도값이 길게 이어지는 구간
- 학습에 넣기 애매한 야간/경계 이탈/급조향 데이터

DonkeyCar Data Manager는 이런 데이터를 사람이 빠르게 찾아보고, 원본은 그대로 둔 채 학습용 데이터만 따로 만드는 것을 목표로 합니다.

## 주요 기능

### 1. 데이터 확인

- DonkeyCar Tub 폴더 로드
- `catalog_*.catalog` 기반 프레임 목록 표시
- 현재 프레임 이미지 미리보기
- 썸네일 타임라인 표시
- 이전/다음/처음/마지막 프레임 이동
- 자동 재생과 재생 배속 조절
- 방향값, 속도값, 주행 방식, 검토 힌트 표시
- 이미지 위에 조향 방향과 주행 상태 오버레이 표시

### 2. 데이터 정리

- 방향값 범위 필터
- 속도값 범위 필터
- 주행 방식/상황 필터
- 이상 후보 자동 표시
- 선택 프레임 제외
- 제외 프레임 복원
- 제외 상태 저장
- 저장하지 않은 정리 상태가 있을 때 종료 경고

### 3. Clean 데이터셋 생성

원본 Tub 폴더는 훼손하지 않고, 학습에 사용할 Clean 폴더를 별도로 생성합니다.

처리 흐름:

1. 원본 Tub 폴더 전체 복사
2. 제외된 이미지 파일 삭제
3. `catalog_*.catalog`에서 제외 프레임 레코드 제거
4. Clean 폴더를 학습 데이터로 사용할지 확인

이 흐름은 발표에서 강조하기 좋습니다.

> 원본 데이터는 보존하고, 학습용 데이터만 안전하게 분리해서 만든다.

### 4. 학습 실행

- WSL 환경 자동 감지
- Conda 경로 자동 감지
- DonkeyCar Python 환경명 설정
- Donkey 프로젝트 폴더 설정
- Tub 경로 설정
- 모델 저장 경로 설정
- 학습 횟수 설정
- 학습 로그 실시간 표시
- 학습 진행률, 현재 학습 횟수, 점수 표시
- 학습 중지 지원

기본적으로 로컬 PC에서는 WSL + Conda + DonkeyCar CLI를 사용합니다.

### 5. 그래프/통계

- 방향값과 속도값 흐름 그래프
- 현재 프레임 위치 표시
- 필터/제외 상태를 반영한 데이터 흐름 확인
- 급격한 조향 변화나 비정상 구간을 시각적으로 확인

### 6. 파일럿 비교

두 개의 학습 모델을 같은 프레임 기준으로 비교하기 위한 탭입니다.

- 파일럿 A/B 모델 선택
- 모델 타입 선택
- 마지막 선택 모델 기억
- 같은 프레임 이미지 좌우 비교
- 밝기 증강 슬라이더
- 블러 증강 슬라이더
- `pilot/angle`, `pilot/throttle` 비교 표시
- `user/angle`, `user/throttle` 기준값 표시

현재는 UI와 설정 저장 구조 중심입니다. 실제 Keras 모델 추론 연결은 이후 확장 작업으로 분리하는 것이 좋습니다.

## 실행 방법

### 요구 환경

- Windows
- Visual Studio 2022 이상 권장
- .NET 10 SDK
- WSL2
- Miniconda 또는 Anaconda
- DonkeyCar가 설치된 Conda 환경

### 실행

Visual Studio에서 실행:

1. `TeamApp.slnx` 또는 `TeamApp.csproj` 열기
2. NuGet 패키지 복원
3. `F5` 실행

터미널에서 실행:

```powershell
dotnet run --project .\TeamApp.csproj
```

빌드만 확인:

```powershell
dotnet build .\TeamApp.csproj
```

## 기본 사용 흐름

### 데이터 정리

1. `데이터 확인` 탭으로 이동
2. `데이터 폴더 열기` 클릭
3. DonkeyCar Tub 폴더 선택
4. 프레임 목록, 이미지, 그래프 확인
5. 이상 후보 또는 필터로 문제 프레임 탐색
6. `선택 프레임 제외` 또는 `복원` 사용
7. `상태 저장` 클릭
8. `클린 폴더 추출` 클릭
9. Clean 폴더를 학습 데이터로 사용

### 학습 실행

1. `학습 실행` 탭으로 이동
2. 자동 감지 결과 확인
3. Tub 경로 확인
4. 모델 저장 경로 확인
5. 학습 횟수 입력
6. `학습 시작` 클릭
7. 로그와 진행률 확인

### 파일럿 비교

1. `파일럿 비교` 탭으로 이동
2. 파일럿 A 모델 선택
3. 파일럿 B 모델 선택
4. 같은 프레임에서 이미지와 출력값 비교
5. 밝기/블러 증강을 켜고 반응 차이 확인

## 폴더 구조

```text
TeamApp/
├─ Form1.cs              # 주요 화면 로직, 데이터 처리, 학습 실행, 그래프 처리
├─ Form1.Designer.cs     # WinForms 디자이너 기반 컨트롤 선언/초기 배치
├─ Form1.resx            # WinForms 리소스
├─ Program.cs            # 앱 시작점
├─ TeamApp.csproj        # 프로젝트 설정
├─ TeamApp.slnx          # 솔루션 파일
├─ resource/             # 폰트 등 UI 리소스
├─ python/               # Python 연동 보조 파일 위치
├─ README.md             # 프로젝트 설명 문서
├─ bin/                  # 빌드 결과물
└─ obj/                  # 빌드 중간 파일
```

## 데이터 폴더 예시

DonkeyCar Tub V2 기준으로 다음과 같은 파일이 필요합니다.

```text
data/
├─ catalog_0.catalog
├─ catalog_1.catalog
├─ manifest.json
└─ images/
   ├─ 0_cam_image_array_.jpg
   ├─ 1_cam_image_array_.jpg
   └─ ...
```

## 저장되는 설정

앱은 일부 설정을 사용자 앱 데이터 폴더에 저장합니다.

| 설정 | 설명 |
| --- | --- |
| `training_settings.json` | WSL, Conda, Donkey 프로젝트, Tub, 모델 저장 경로 |
| `pilot_arena_settings.json` | 파일럿 비교 탭에서 마지막으로 선택한 두 모델 |
| `deleted_frames_meta.txt` | Tub 폴더 안에 저장되는 제외 프레임 목록 |

## 문제 해결

### Tub 폴더를 읽지 못함

확인할 것:

- `catalog_*.catalog` 파일이 있는지
- `manifest.json` 파일이 있는지
- 이미지 파일이 루트 또는 `images/` 폴더 안에 있는지
- 경로에 접근 권한이 있는지

### 학습 시작 버튼이 동작하지 않음

확인할 것:

- WSL이 설치되어 있는지
- Conda 경로가 맞는지
- DonkeyCar 환경명이 맞는지
- Donkey 프로젝트 폴더에 `manage.py` 또는 `config.py`가 있는지
- Tub 경로가 Windows 경로에서 WSL 경로로 변환 가능한지

### Epoch가 설정대로 적용되지 않음

DonkeyCar CLI 버전에 따라 `--epochs` 옵션을 직접 받지 않을 수 있습니다. 이 앱은 임시 config 파일에 `MAX_EPOCHS` 값을 넣어 학습 횟수를 제어하는 방식으로 처리합니다.

### NU1701 경고가 보임

현재 `OpenTK`, `OpenTK.GLControl`, `SkiaSharp.Views.WindowsForms` 패키지에서 `NU1701` 경고가 보일 수 있습니다. 빌드는 가능하지만, 패키지가 .NET 10 대상에 완전히 맞지 않는다는 경고입니다.

이 경고는 당장 실행을 막는 오류는 아닙니다. 패키지 교체나 프로젝트 대상 프레임워크 조정은 별도 작업으로 분리하는 것이 안전합니다.

## 발표 포인트

- 자율주행 알고리즘 개발이 아니라 **데이터 관리 도구**에 집중
- 원본 데이터 보존
- 학습용 Clean 데이터셋 별도 생성
- 조향값/속도값 기반 이상 데이터 탐색
- 학습 로그와 진행 상태 표시
- 두 모델 비교를 위한 Pilot Arena 확장 가능성

## 향후 개선 계획

- 파일럿 비교 탭에 실제 Keras 모델 추론 연결
- 데이터 정리 결과 리포트 자동 생성
- 제외된 구간을 타임라인에서 더 명확하게 표시
- Tub별 통계 요약 카드 추가
- 학습 결과 모델별 점수/로그 히스토리 관리
- 발표용 데모 데이터와 샘플 영상 추가
