# DonkeyCar Data Manager

DonkeyCar 주행 데이터를 확인하고, 문제 프레임을 정리하고, 정리된 데이터로 학습한 뒤, 학습된 모델을 다시 검증할 수 있는 C# WinForms 기반 데이터 관리 도구입니다.

이 프로젝트의 핵심 목표는 자율주행 알고리즘을 새로 만드는 것이 아니라, DonkeyCar가 학습할 데이터셋을 더 안전하고 이해하기 쉽게 관리하는 것입니다.

---

## 프로젝트 방향

DonkeyCar는 주행 중 이미지와 조향값, 속도값, 주행 모드 같은 데이터를 Tub 폴더에 저장합니다. 이 데이터에는 정상 주행뿐 아니라 차선 이탈, 벽 충돌 직전, 정지에 가까운 속도, 잘못된 조향 입력 같은 학습에 방해되는 프레임이 섞일 수 있습니다.

DonkeyCar Data Manager는 사용자가 이런 문제 프레임을 직접 확인하고 제외한 뒤, 원본은 보존한 상태로 학습용 Clean 폴더를 만들어 학습까지 이어갈 수 있도록 돕습니다.

주요 흐름은 다음과 같습니다.

```text
Tub 데이터 열기
-> 프레임 이미지와 조향/속도 확인
-> 이상 후보 또는 직접 선택으로 문제 프레임 제외
-> Clean 폴더 추출
-> DonkeyCar 모델 학습
-> 추론/검증 탭에서 실제값과 예측값 비교
```

## 1. 데이터 확인 탭

데이터 확인 탭은 프로그램의 중심 화면입니다. DonkeyCar Tub 폴더를 열고 프레임 단위로 데이터를 확인합니다.

### 데이터 폴더 열기

`데이터 폴더 열기` 버튼으로 DonkeyCar Tub 폴더를 선택합니다. 프로그램은 폴더 안의 `catalog_*.catalog` 파일을 읽어 프레임 목록을 구성합니다.

지원하는 데이터 구조 예시는 다음과 같습니다.

```text
data/
|-- catalog_0.catalog
|-- catalog_1.catalog
|-- manifest.json
|-- images/
    |-- 0_cam_image_array_.jpg
    |-- 1_cam_image_array_.jpg
```

### 프레임 목록

왼쪽 목록에는 이미지 파일명이 표시됩니다. 너무 많은 정보를 한 번에 보여주지 않도록 현재는 프레임 식별에 필요한 이미지명 중심으로 구성되어 있습니다.

프레임을 선택하면 다음 정보가 함께 갱신됩니다.

| 표시 정보 | 의미 |
| --- | --- |
| 이미지 미리보기 | 선택한 프레임의 카메라 이미지 |
| 방향값 | DonkeyCar의 조향값, 음수는 좌회전, 양수는 우회전 |
| 속도값 | 전진 또는 정지에 가까운 정도를 나타내는 값 |
| 주행 방식 | 사용자 조작, 로컬 주행 등 주행 모드 |
| 검토 상태 | 급조향, 정지에 가까운 속도 등 검토 필요 여부 |

### 이미지 오버레이

미리보기 이미지 위에는 현재 프레임의 주행 방향을 시각적으로 보여주는 오버레이가 표시됩니다.

오버레이 구성:

- 주행 상태 텍스트
- 조향 방향 화살표
- 조향/속도 상태를 빠르게 이해할 수 있는 시각 표시

이 기능은 발표나 데모에서 “이 프레임이 왜 정상 또는 이상 후보인지” 설명할 때 유용합니다.

### 썸네일 타임라인

프레임을 빠르게 훑어볼 수 있도록 썸네일 타임라인이 제공됩니다. 현재 선택된 프레임은 테두리로 강조됩니다.

활용 예시:

- 차선 이탈이 시작되는 구간 찾기
- 충돌 직전 구간 찾기
- 특정 커브 구간을 연속으로 확인하기

### 자동 재생과 배속

자동 재생 버튼을 누르면 프레임이 순서대로 재생됩니다. 배속은 1배부터 10배까지 선택할 수 있습니다.

빠르게 전체 데이터를 훑어볼 때는 5배 이상을 사용하고, 문제 구간을 자세히 볼 때는 1배 또는 2배를 사용하는 것이 좋습니다.

## 2. 데이터 정리 기능

데이터 정리는 학습에 사용하면 안 되는 프레임을 제외하고, 필요한 경우 다시 복원하는 기능입니다.

### 필터 검색

다음 조건으로 프레임을 좁혀 볼 수 있습니다.

| 필터 | 설명 |
| --- | --- |
| 방향값 범위 | 조향값 기준으로 좌회전/우회전/직진 구간 필터링 |
| 속도값 범위 | 정지에 가까운 구간, 빠른 전진 구간 필터링 |
| 주행 방식 | 사용자 조작, 로컬 주행 등 모드 기준 필터링 |
| 상황 | normal, out_of_bound 같은 시나리오 기준 필터링 |

예를 들어 차가 직진 화면인데 조향값이 강한 좌회전 또는 우회전으로 기록된 구간은 필터로 찾아 검토할 수 있습니다.

### 이상 후보 보기

`이상 후보 보기` 버튼은 자동 기준으로 검토가 필요한 프레임만 보여줍니다.

후보 기준 예시:

- 조향값이 급격하게 변하는 프레임
- 속도값이 정지에 가까운 프레임
- 시나리오가 `out_of_bound`인 프레임
- catalog 값이 누락되었거나 비정상적인 프레임

이 기능은 사용자가 모든 프레임을 수동으로 확인하지 않아도 문제 가능성이 높은 구간부터 볼 수 있도록 돕습니다.

### 선택 프레임 제외

`선택 프레임 제외`는 현재 선택된 프레임을 학습에서 제외합니다.

중요한 점:

- 원본 이미지 파일을 바로 삭제하지 않습니다.
- 내부 상태에서만 제외 표시합니다.
- 나중에 복원할 수 있습니다.
- 저장하지 않은 제외 상태가 있으면 종료 또는 데이터 변경 시 경고합니다.

### 복원

`복원` 버튼은 제외 처리된 프레임을 다시 학습 사용 상태로 되돌립니다. 잘못 제외한 프레임을 복구할 때 사용합니다.

### 상태 저장

`상태 저장` 버튼은 제외한 프레임 정보를 `deleted_frames_meta.txt`로 저장합니다.

저장 파일은 Tub 폴더 안에 생성되며, 같은 폴더를 다시 열면 이전 제외 상태를 자동으로 복원합니다.

```text
data/
|-- deleted_frames_meta.txt
```

### Clean 폴더 추출

`클린 폴더 추출`은 발표에서 가장 중요한 기능 중 하나입니다.

처리 방식:

1. 원본 Tub 폴더를 그대로 복사합니다.
2. 제외 표시된 이미지 파일을 복사본에서 제거합니다.
3. `catalog_*.catalog` 파일에서도 제외된 프레임 기록을 제거합니다.
4. 원본 폴더는 절대 직접 훼손하지 않습니다.
5. 정리된 Clean 폴더를 학습용 데이터로 사용할 수 있습니다.

발표 설명 예시:

```text
원본 데이터는 보존하고,
학습에 방해되는 프레임만 제외한 Clean 폴더를 새로 만들어
AI가 더 좋은 문제집으로 학습하도록 돕습니다.
```

## 3. 그래프/통계 탭

그래프/통계 탭은 전체 주행 데이터의 방향값과 속도값 흐름을 보여줍니다.

### 표시 항목

| 그래프 요소 | 설명 |
| --- | --- |
| Steering | 조향값 변화 |
| Speed | 속도값 변화 |
| Zero | 0 기준선 |
| Playhead | 현재 프레임 위치 |
| Selected | 선택 또는 제외 구간 표시 |

### 활용 방법

그래프를 보면 이미지 하나씩 확인하지 않아도 데이터 흐름을 빠르게 파악할 수 있습니다.

확인하기 좋은 패턴:

- 조향값이 갑자기 튀는 구간
- 속도값이 오래 0에 가까운 구간
- 좌우 조향이 비정상적으로 반복되는 구간
- 차선 이탈이 의심되는 구간
- 제외한 프레임이 전체 흐름에서 어느 위치인지

## 4. 학습 실행 탭

학습 실행 탭은 DonkeyCar 학습 명령을 WinForms 안에서 실행하고 로그를 확인하는 화면입니다.

### 자동 감지

`자동 감지`는 현재 PC의 학습 환경을 찾습니다.

감지 대상:

| 항목 | 예시 |
| --- | --- |
| WSL 배포판 | `Ubuntu-22.04` |
| Conda 경로 | `/home/user/miniconda3/bin/conda` |
| Python 환경명 | `e2e_env`, `donkey53` |
| DonkeyCar 프로젝트 경로 | `/home/user/mycar`, `/home/user/mysim` |

자동 감지가 실패하면 사용자가 직접 경로와 환경명을 입력할 수 있습니다.

### 학습 설정

| 설정 | 설명 |
| --- | --- |
| Donkey 프로젝트 경로 | `manage.py`, `config.py`가 있는 DonkeyCar 작업 폴더 |
| Tub 경로 | 학습에 사용할 데이터 폴더 |
| 모델 저장 경로 | 학습 결과 모델이 저장될 위치 |
| 모델 종류 | DonkeyCar 모델 타입, 예: `linear` |
| Python 환경명 | Conda 환경 이름 |
| 학습 횟수 | 학습 epoch 수 |

모델 저장 경로는 기본적으로 사용자 문서 폴더 아래 `donkeycar_models` 하위에 생성됩니다. 데이터 폴더명 기준으로 모델 폴더를 나누어 관리할 수 있습니다.

예시:

```text
C:\Users\user\Documents\donkeycar_models\data_codex_fresh_minimonaco_5k\pilot.keras
```

### 학습 로그

학습 로그는 DonkeyCar 학습 과정에서 출력되는 내용을 보여줍니다.

사용자가 보기 불편한 원본 로그를 그대로 쏟아내지 않고, 진행률과 학습 상태를 더 보기 쉽게 요약하도록 개선되어 있습니다.

표시 항목:

- 학습 상태
- 현재 학습 횟수
- 진행률
- 점수
- 종료 코드
- 조기 종료 이유

### 학습 점수

점수는 사용자가 loss 값을 직접 해석하지 않아도 학습 상태를 대략 이해할 수 있도록 만든 요약값입니다.

주의:

- 점수는 절대적인 모델 성능 지표가 아닙니다.
- 같은 데이터셋과 같은 조건에서 모델끼리 비교할 때 참고용으로 보는 값입니다.
- 실제 성능 판단은 추론/검증 탭에서 실제값과 예측값을 비교해야 합니다.

### 조기 종료 안내

DonkeyCar 학습은 설정한 학습 횟수보다 빨리 끝날 수 있습니다. 예를 들어 100회로 설정했는데 20회 부근에서 종료될 수 있습니다.

가능한 이유:

- 검증 loss가 더 이상 개선되지 않음
- Early Stopping 조건이 만족됨
- 모델이 충분히 수렴했다고 판단됨
- 사용자가 학습 중지를 누름
- Python/환경 오류로 종료됨

프로그램은 가능한 경우 학습 완료 메시지에 왜 일찍 끝났는지 설명합니다.

### 실시간 학습 그래프

학습 중에는 train loss와 validation loss를 그래프로 확인할 수 있습니다.

그래프 해석:

| 패턴 | 의미 |
| --- | --- |
| train/validation 둘 다 감소 | 학습이 정상적으로 진행 중일 가능성이 높음 |
| train만 감소, validation 증가 | 과적합 가능성 |
| 둘 다 거의 변하지 않음 | 데이터 또는 학습 설정 점검 필요 |
| 매우 빠르게 0 근처로 수렴 | 데이터가 단순하거나 검증 필요 |

## 5. 추론/검증 탭

추론/검증 탭은 학습된 모델이 실제 프레임에 대해 어떤 조향값과 속도값을 예측하는지 확인하는 화면입니다.

### 사용 순서

```text
1. 데이터 폴더 선택
2. 모델 파일 선택
3. 자동 감지
4. 서버 시작
5. 프레임 검증
```

### 데이터 폴더와 모델 파일 선택

학습 실행 탭에서 학습이 완료되면 모델 경로가 추론/검증 탭에 자동 연결될 수 있습니다. 직접 다른 모델을 선택해서 비교하는 것도 가능합니다.

### Python 추론 서버

추론/검증은 Python 서버를 통해 모델 예측값을 받아옵니다.

포함된 서버 파일:

```text
python/pilot_inference_server.py
```

서버 시작 전 확인하는 항목:

- 모델 파일이 실제로 존재하는지
- 데이터 폴더가 존재하는지
- WSL/Conda 환경이 준비되어 있는지
- Python 의존성이 설치되어 있는지
- 서버 포트가 사용 가능한지

### 실제값과 예측값 비교

선택한 프레임에 대해 다음 값을 비교합니다.

| 항목 | 설명 |
| --- | --- |
| 실제 Angle | 데이터셋에 기록된 사용자 조향값 |
| 예측 Angle | 모델이 이미지로부터 예측한 조향값 |
| 실제 Throttle | 데이터셋에 기록된 속도값 |
| 예측 Throttle | 모델이 예측한 속도값 |

이미지 위에는 실제 방향과 예측 방향을 함께 표시하여 모델이 실제 데이터와 비슷하게 반응하는지 확인할 수 있습니다.

### 예측 차이 후보

`예측 차이 후보` 기능은 실제값과 예측값의 차이가 큰 프레임을 자동으로 모아 검토할 수 있게 합니다.

활용 예시:

- 실제는 좌회전인데 모델은 우회전으로 예측하는 구간 찾기
- 특정 구간에서 예측값이 한쪽 방향으로 고정되는 문제 찾기
- 학습 데이터 품질 문제인지 모델 학습 문제인지 확인하기

### 자동 재생과 배속

추론/검증 탭에서도 처음, 이전, 다음, 끝 이동 버튼과 자동 재생, 배속 선택을 제공합니다.

여러 프레임을 연속으로 보면서 모델 예측이 실제 주행 흐름을 따라가는지 확인할 수 있습니다.

## 6. 발표 데모 추천 흐름

교수님 피드백 방향에 맞춰 다음 흐름으로 데모를 준비하는 것이 좋습니다.

### 데모 1: 원본 데이터 확인

1. 데이터 확인 탭에서 Tub 폴더를 엽니다.
2. 자동 재생으로 주행 데이터를 훑어봅니다.
3. 차선 이탈 또는 이상 조향 구간을 찾습니다.

### 데모 2: 문제 프레임 제외

1. 이상 후보 보기를 누릅니다.
2. 후보 프레임을 이미지와 그래프로 확인합니다.
3. 선택 프레임 제외를 누릅니다.
4. 제외 상태를 저장합니다.

### 데모 3: Clean 폴더 생성

1. 클린 폴더 추출을 누릅니다.
2. 원본은 보존되고 Clean 폴더만 생성된다는 점을 설명합니다.
3. Clean 폴더를 학습 데이터로 사용합니다.

### 데모 4: 학습 실행

1. 학습 실행 탭에서 자동 감지를 실행합니다.
2. Clean 폴더를 Tub 경로로 지정합니다.
3. 학습 시작을 누릅니다.
4. 진행률, 점수, 학습 그래프를 보여줍니다.

### 데모 5: 추론/검증

1. 학습된 모델을 추론/검증 탭에 연결합니다.
2. 서버를 시작합니다.
3. 실제값과 예측값을 비교합니다.
4. 예측 차이가 큰 후보를 보여줍니다.

## 자주 발생하는 문제

### 한글이 깨져 보일 때

소스 파일과 README는 UTF-8 기준으로 관리합니다. Visual Studio에서 파일을 열 때 인코딩이 깨져 보이면 UTF-8로 다시 열어야 합니다.

### Tub 폴더가 열리지 않을 때

다음을 확인합니다.

- `catalog_*.catalog` 파일이 있는지
- 이미지 파일이 존재하는지
- 폴더 접근 권한이 있는지
- 데이터 구조가 DonkeyCar Tub 형식과 맞는지

### 학습 시작 버튼이 동작하지 않을 때

다음을 확인합니다.

- WSL이 설치되어 있는지
- Conda가 설치되어 있는지
- DonkeyCar가 설치된 Conda 환경이 있는지
- Donkey 프로젝트 경로에 `manage.py` 또는 `config.py`가 있는지
- Tub 경로가 실제로 존재하는지

### 추론 서버가 시작되지 않을 때

다음을 확인합니다.

- 모델 파일 경로가 올바른지
- Python 추론 서버 파일이 복사되어 있는지
- 서버 포트가 이미 사용 중인지
- Conda 환경에 필요한 Python 패키지가 설치되어 있는지

### 모델이 한쪽 방향만 예측할 때

가능한 원인:

- 학습 데이터가 한쪽 방향으로 치우침
- 학습 횟수가 부족하거나 너무 빨리 조기 종료됨
- Clean 폴더 생성 과정에서 필요한 데이터가 과도하게 제외됨
- 모델 타입이나 설정이 데이터와 맞지 않음

확인 방법:

1. 그래프/통계 탭에서 조향값 분포를 확인합니다.
2. 추론/검증 탭에서 실제값과 예측값 차이를 확인합니다.
3. 예측 차이 후보를 확인합니다.
4. 원본 데이터와 Clean 데이터를 비교합니다.

## 현재 한계와 개선 예정

- 예측 차이 후보 기준값을 사용자가 직접 조절하는 UI가 필요합니다.
- 학습 전 데이터 분포 검사 기능이 더 강화되면 좋습니다.
- 조기 종료 설정을 UI에서 직접 켜고 끄는 기능이 필요합니다.
- 추론 서버 포트 충돌 시 자동으로 대체 포트를 찾는 기능이 필요합니다.
- 데이터 정리 결과를 PDF 또는 HTML 리포트로 내보내는 기능이 있으면 발표 자료 준비가 쉬워집니다.
- 튜토리얼 내용을 추론/검증 탭 중심으로 더 보강할 필요가 있습니다.
