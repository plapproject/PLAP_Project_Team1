# DonkeyCar Data Manager

> DonkeyCar 주행 데이터 관리, 데이터 정제, 학습 실행, 모델 비교를 하나의 화면에서 지원하는 C# WinForms 기반 통합 UI

---

## 프로젝트 소개

**DonkeyCar Data Manager**는 DonkeyCar 자율주행 프로젝트에서 생성되는 주행 데이터인 **Tub 데이터**를 효율적으로 확인하고 정리하기 위해 만든 데이터 관리 도구입니다.

DonkeyCar로 주행 데이터를 수집하면 정상 데이터뿐 아니라 충돌, 차선 이탈, 잘못된 조향 입력, 정지에 가까운 구간처럼 학습에 넣기 애매한 데이터가 함께 쌓입니다. 이 프로젝트는 이런 데이터를 사용자가 직접 확인하고, 문제 프레임을 제외하고, 원본을 보존한 상태로 학습용 Clean 데이터셋을 만들 수 있도록 지원합니다.

또한 WSL/Conda 기반 DonkeyCar 학습 실행 기능을 제공하여 복잡한 터미널 명령어를 직접 입력하지 않아도 UI 안에서 학습을 시작하고 로그를 확인할 수 있습니다.

> 핵심 방향: **AI 모델 자체를 새로 개발하는 프로그램이 아니라, AI가 학습할 데이터를 더 잘 관리하기 위한 전문 도구**

---

## 개발 목표

- DonkeyCar Tub 데이터 통합 관리
- 주행 프레임 이미지와 조향/속도 데이터 시각화
- 이상 데이터 후보 탐색 및 제외/복원 지원
- 원본을 훼손하지 않는 Clean 학습 데이터셋 생성
- WSL/Conda 기반 DonkeyCar 학습 실행
- 학습 로그와 진행 상태 확인
- 초보자도 사용할 수 있는 WinForms GUI 제공
- 두 모델의 반응을 비교할 수 있는 Pilot Arena 확장

## 주요 기능

### 1. 데이터 확인

DonkeyCar Tub 폴더를 불러와 프레임 단위로 데이터를 확인하는 기능입니다.

주요 기능:

- DonkeyCar Tub 폴더 열기
- 여러 `catalog_*.catalog` 파일 로드
- 프레임 목록 표시
- 현재 프레임 이미지 미리보기
- 썸네일 바를 통한 빠른 프레임 탐색
- 이전/다음/처음/마지막 프레임 이동
- 자동 재생
- 재생 배속 조절
- 현재 프레임의 방향값, 속도값, 주행 방식 표시
- 이미지 위에 조향 방향과 주행 상태 오버레이 표시

사용자가 확인할 수 있는 정보:

| 항목 | 설명 |
| --- | --- |
| 이미지 파일 | 현재 프레임의 카메라 이미지 |
| 방향값 | 좌회전/우회전/직진 정도 |
| 속도값 | 전진/정지/후진 정도 |
| 주행 방식 | 사용자 조작, AI 주행 등 |
| 검토 힌트 | 급조향, 정지에 가까운 속도 등 확인 필요 여부 |

추천 시연 자료:

```md
![데이터 확인 탭](docs/media/data-viewer.png)
```

> 위 이미지에는 프레임 목록, 이미지 미리보기, 썸네일 바, 조향/속도 그래프가 함께 보이도록 캡처하면 좋습니다.

### 2. 데이터 정리

학습에 사용하기 어려운 프레임을 제외하거나 다시 복원하는 기능입니다.

주요 기능:

- 방향값 범위 필터
- 속도값 범위 필터
- 주행 방식 필터
- 상황 필터
- 이상 후보 보기
- 선택 프레임 제외
- 제외 프레임 복원
- 제외 상태 저장
- 저장하지 않은 정리 상태가 있을 때 종료 경고

데이터 정리 예시:

| 상황 | 처리 방법 |
| --- | --- |
| 차선 이탈 장면 | 해당 프레임 제외 |
| 벽 충돌 직전/직후 | 구간 단위로 선택 후 제외 |
| 이미지와 조향값이 맞지 않음 | 이상 후보로 확인 후 제외 |
| 실수로 제외함 | 복원 버튼으로 다시 학습 사용 상태로 변경 |

데이터 정리는 **소프트 삭제 방식**을 기본으로 합니다.

- 원본 파일은 바로 삭제하지 않음
- 제외 상태만 저장
- 필요하면 복원 가능
- 최종 학습용 Clean 폴더를 만들 때만 제외 프레임을 제거

추천 시연 자료:

```md
![데이터 정리 흐름](docs/media/cleanup-flow.gif)
```

> GIF로 `이상 후보 보기 -> 프레임 선택 -> 선택 프레임 제외 -> 상태 저장` 흐름을 보여주면 발표에서 이해가 쉽습니다.

### 3. Clean 학습용 폴더 추출

원본 Tub 데이터를 보존하면서 학습에 사용할 Clean 데이터셋을 따로 만드는 기능입니다.

처리 과정:

1. 현재 Tub 폴더 전체 복사
2. 제외 상태인 이미지 제거
3. `catalog_*.catalog`에서 제외 프레임 레코드 제거
4. Clean 폴더 생성 완료
5. 생성한 Clean 폴더를 학습 데이터로 사용할지 확인

장점:

- 원본 데이터 보존
- 학습용 데이터만 분리
- 실수로 원본을 훼손할 위험 감소
- 발표 시 데이터 관리 도구의 차별점으로 설명하기 좋음

발표 문장 예시:

> 이 프로그램은 원본 데이터를 직접 삭제하지 않고, 사용자가 제외 표시한 프레임만 제거한 Clean 학습 폴더를 따로 생성합니다.

### 4. 그래프/통계

주행 데이터의 흐름을 그래프로 확인하는 기능입니다.

주요 기능:

- 방향값 그래프 표시

### 5. 학습 실행

DonkeyCar 학습 명령을 UI에서 실행하고 결과를 확인하는 기능입니다.

주요 기능:

- WSL 환경 자동 감지
- Conda 경로 자동 감지
- DonkeyCar Python 환경명 설정
- Donkey 프로젝트 경로 설정
- Tub 경로 설정
- 모델 저장 경로 설정
- 학습 횟수 설정
- 학습 시작
- 학습 중지
- 학습 로그 실시간 출력
- 진행률, 학습 횟수, 점수 표시

학습 실행 흐름:

1. 학습 데이터 경로 선택
2. 모델 저장 경로 선택
3. WSL/Conda/DonkeyCar 환경 확인
4. 학습 횟수 입력
5. 학습 시작
6. 로그와 진행 상태 확인
7. 모델 저장 확인

자동 감지 대상:

| 항목 | 설명 |
| --- | --- |
| WSL 배포판 | Ubuntu 등 WSL 환경 |
| Conda 경로 | `~/miniconda3/bin/conda` 등 |
| Python 환경명 | `e2e_env`, `donkey53` 등 |
| Donkey 프로젝트 | `~/mycar`, `~/mysim` 등 |
| Tub 경로 | 학습에 사용할 데이터 폴더 |

## 문제 해결

### Tub 폴더가 열리지 않을 때

확인할 내용:

- `catalog_*.catalog` 파일이 있는지
- `manifest.json` 파일이 있는지
- 이미지가 루트 또는 `images/` 폴더에 있는지
- 폴더 접근 권한이 있는지

### 학습 실행이 안 될 때

확인할 내용:

- WSL이 설치되어 있는지
- Conda가 설치되어 있는지
- DonkeyCar가 설치된 Python 환경이 있는지
- Donkey 프로젝트 폴더에 `manage.py` 또는 `config.py`가 있는지
- Tub 경로가 실제로 존재하는지

### 자동 감지가 실패할 때

자동 감지는 다음 이름들을 우선 탐색합니다.

- Donkey 프로젝트: `~/mycar`, `~/mysim`
- Conda 환경: `e2e_env`, `donkey53`
- Conda 경로: `~/miniconda3/bin/conda`

자동 감지가 실패하면 사용자가 직접 경로와 환경명을 입력할 수 있습니다.

### NU1701 경고가 보일 때

`OpenTK`, `OpenTK.GLControl`, `SkiaSharp.Views.WindowsForms` 패키지에서 .NET 10과의 호환성 경고가 발생할 수 있습니다.

현재는 빌드와 실행을 막는 오류는 아니므로, 패키지 교체는 별도 작업으로 분리하는 것이 좋습니다.

## 발표 포인트

- 자율주행 알고리즘 개발이 아니라 **데이터 관리 도구**에 집중
- 원본 데이터는 보존하고 Clean 학습 데이터만 별도 생성
- 조향값/속도값 기반 이상 데이터 탐색
- 학습 실행과 로그 확인을 GUI에서 수행
- 파일럿 비교 기능으로 모델 개선 여부를 시각적으로 비교 가능
