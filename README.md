# Donkeycar Data Manager

Donkeycar Data Manager는 DonkeyCar Tub 데이터를 불러와 프레임을 확인하고, 잘못된 데이터를 제외/복원한 뒤 학습용 Clean 폴더로 추출하는 WinForms 기반 데이터 관리 도구입니다.

## 실행 방법

1. Visual Studio에서 `TeamApp.csproj` 또는 솔루션 파일을 엽니다.
2. NuGet 패키지 복원을 진행합니다.
3. `F5`로 실행하거나 아래 명령을 사용합니다.

```powershell
dotnet run --project .\TeamApp.csproj
```

## 주요 기능

- DonkeyCar Tub 폴더 및 다중 `catalog_*.catalog` 로드
- 프레임 이미지, 조향값, 스로틀값, 모드, 시나리오 확인
- 조향값/스로틀값/모드/시나리오 기준 필터링
- 선택 프레임 제외, 구간 제외, 구간 복원
- 제외/복원 상태를 `deleted_frames_meta.txt`로 저장 및 자동 복원
- 저장하지 않은 정리 작업이 있을 때 상태바/창 제목 표시 및 종료 경고
- 원본을 훼손하지 않는 학습용 `_Clean` 폴더 추출
- 조향값/스로틀값 그래프 표시
- WSL/conda 기반 DonkeyCar 학습 실행 및 로그 관찰
- 핵심 작업 흐름 튜토리얼

## 폴더 구조

```text
TeamApp/
├─ Form1.cs              # 주요 화면 로직과 데이터 처리
├─ Form1.Designer.cs     # WinForms UI 배치 코드
├─ Form1.resx            # WinForms 리소스
├─ Program.cs            # 프로그램 시작점
├─ TeamApp.csproj        # 프로젝트 설정
├─ README.md             # 프로젝트 설명 문서
├─ bin/                  # 빌드 출력물
└─ obj/                  # 빌드 중간 산출물
```

## 사용 방법

1. `데이터 폴더 열기`로 Tub 폴더를 선택합니다.
2. 프레임 표와 미리보기 이미지에서 데이터를 확인합니다.
3. `이상 후보 보기` 또는 `필터 적용`으로 확인할 프레임을 좁힙니다.
4. 문제가 있는 데이터는 `선택 프레임 제외` 또는 `구간 제외`로 학습 대상에서 제외합니다.
5. 실수로 제외한 데이터는 `복원`으로 되돌립니다.
6. 작업 중 상태바의 `저장 필요` 표시를 확인하고 `상태 저장`을 누릅니다.
7. 최종 학습 데이터가 필요하면 `클린 폴더 추출`을 눌러 원본 옆에 `_Clean` 폴더를 생성합니다.
8. 학습 실행 탭에서 Tub 경로, mycar 경로, 모델 저장 경로 등을 입력한 뒤 학습을 실행합니다.

## 에러 해결 방법

- 빌드 중 `TeamApp.exe`를 복사할 수 없다는 오류가 나오면 실행 중인 프로그램을 종료한 뒤 다시 빌드합니다.
- Tub 폴더를 열었는데 데이터가 없으면 `catalog_*.catalog` 파일과 이미지 파일이 있는지 확인합니다.
- 이미지가 보이지 않으면 catalog의 이미지명과 실제 파일명이 일치하는지 확인합니다.
- 필터 결과가 0개면 조향값/스로틀값 범위와 모드/시나리오 조건을 넓혀 봅니다.
- 학습 실행 오류가 나면 WSL, conda 경로, Python 환경명, mycar 경로가 맞는지 확인합니다.
- `NU1701` 경고는 일부 NuGet 패키지 호환성 경고입니다. 현재 빌드를 막는 오류는 아니지만 추후 패키지 업데이트가 필요할 수 있습니다.
